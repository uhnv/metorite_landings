Meteorite Data Representation

The goal of this project is to demonstrate the ability to:

  • Consume data programmatically from a given resource
  • Understand the data presented
  • Organize/disseminate/communicate the data in a presentable format
  • Utilize a GitHub source code repository
The data is taken from NASA meteorite landings "Downloads & Resources" page.

In addition to requirements this project is made using WSU Colors Guidance.

Consuming and Analyzing Data

Data visualization and analysis were done in two parts of the project. In the navigation bar, the user can go to "Map visualization" or "Chart representation" to see different ways to consume the provided data.

The "Map visualization" page is a representation of the data from JSON file. This is an interactive map with meteorite landings across the World. To create this meteorite landings map I have used LeafletJS open source interactive map library. Each circle on the map represents a meteorite landing indicating its actual position on the map. Some of the library functionality was used, you can click on the marker and see the pop-up label with additional information for a particular meteorite.

The Chart representation page has a pie chart and a bar chart. The pie chart represents a proportion of the different types of meteorites found on the Earth. The bar chart is a graphical representation of the size categories of the meteorites found on the Earth.

Problems

After the initial setup was done it was clear that not all entries were displayed on the map.
This might be caused by two issues:

  • Leaflet might have limitations on rendering a large number of the objects (markers) on the map.
  • Additionally, not all the entries have valid longitude and latitude. (See the image below. The highlighted area is the entry with no geolocation.) Thus, the code execution stops on the first invalid entry throwing the error in the browser console.

Solutions:

  • Filter out the invalid values, and handle the errors, so the execution will not be interrupted.
  • The limitation issue was solved using the circles instead of markers, here is the solution idea.
It seems that circles worked better than markers, and all entries are processed (see the console output on the image below). Another possible way to solve this issue, and, probably, make performance better might be a plugin for LeafletJS library (not tested yet), or using another library.

Observations

The provided data is not consistent. The issue with rendering all entries from the JSON might not be present if geolocation were present for all entries.
