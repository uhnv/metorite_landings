Meteorite Data Representation

The goal of this project is to demonstrate the ability to:

  • Consume data programmatically from a given resource
  • Understand the data presented
  • Organize/disseminate/communicate the data in a presentable format
  • Utilize a GitHub source code repository
The data is taken from NASA meteorite landings "Downloads & Resources" page.

In addition to requirements this web page is made using WSU Colors Guidance.

Consuming and Analyzing Data

Data analyzing was done in two parts of the project. In the navigation bar you can go to "Map visualization" or "Chart representation" to see different ways to consume provided data.

The "Map visualization" page is the representation of the data from JSON file. This is an interactive map with meteorite landings actross the World. To create this meteorite landings map I have used LeafletJS open source interactive map liblrary. Each marker on the map represents a meteorite landing indicating it's actual position on the map. The variety of the library functionality was used, you can click on the marker and see the pop-up label with additional information for particular meteorite. All data as the marker position and pop-up label is taken from the provided JSON file.

Problems

I. After the initial setup was done I realized that not all entries were displayed on the map.
This might be caused by two issuess:

  • Leaflet might have limitations on rendering large number of the objects (markers) on the map.
  • Additionally, not all the entries have valid longitude and latitude. (See the image below. The higlited stuff is the entry with no geolocation.) Thus, the code execution stops on the first invalid entry throwing the error in the browser console.

Solutions:

  • Filter out the invalid values, and handle the errors, so the execution will not be interrupted.
  • The limitation issue was solved using the cirkles instead of markers, here is the solution idea.
It seems that cirkles worked out, and performanse looks not the best at the same time, but still usable. All entries are processed (see the console output on the image below.) Another possible way to solve this issue, and make performace better might be a plugin for LeafletJS library (not tested yet), or changing the LeafletJS library to another one.

Observations

Provided data is not consistent. The issue with rendering all entries from the JSON might not be present if data would be more accurate.
